## The LAMBDA programming language

The datatype for the LAMBDA programming language,
which has not changed since Week 4, is given as follows.
Note that a completed LAMBDA compiler should handle ALL
the langugage constructs listed in this datatype:

```
datatype dexp =
//
| DEint of sint
| DEbtf of bool
| DEstr of strn
//
| DEvar of dvar
| DElam of
( dvar, dexp(*body*))
| DEapp of
( dexp(*fun*), dexp(*arg*))
//
| DEopr of (strn, list(dexp))
//
| DEfst of (dexp)
| DEsnd of (dexp)
| DEtup of (dexp, dexp)
//
| DEif0 of
( dexp(*test*)
, dexp(*then*), dexp(*else*))
//
| DEfix of
( dvar(*fun*)
, dvar(*arg*), dexp(*body*))
//
| DElet of
( dvar(*x*)
, dexp(*def*), dexp(*scope*))
//
| DElist_nil of ()
| DElist_cons of (dexp, dexp)
//
| DElazy of dexp
| DEstcn_nil of ()
| DEstcn_cons of (dexp, dexp)
//
| DEarry_size$val of (dexp(*size*), dexp(*val*))
| DEarry_size$fun of (dexp(*size*), dexp(*fun*))
//
| DEanno of (dexp, styp(*anno*))
//
| DElam1 of
  (dvar(*x*), styp, dexp(*body*))
| DEfix1 of
  (dvar(*f*), dvar(*x*), styp(*arg*), styp(*res*), dexp(*body*))
//
```

## Five Expected Compiler Components

I expect to see the following 5 (five) components in your
compiler:

1. A closure-based interpreter (dexp_eval000) LAMBDA.
  (Note that there is no need for implementing a substitution-based
  interpreter for LAMBDA).

2. An inference-based type-checker (dexp_oftp000) for LAMBDA

3. A translation function (dexp_comp000) from LAMBDA to some form of
  representation of computations (where a computation consists of a
  sequence of instructions plus a temporary register for storing the
  result of the computation)

4. An emitter function (dcmp_emit000) that emits JavaScript code for
  the computations generated by dexp_comp000. Note that JavaScript is
  the TARGET language here. If you have to use Python instead of JS,
  I will give you at most 50% of the points assigned to this component.

5. Various tests covering ALL the language constructs of LAMBDA.
  In addtion, I expect to see some "large" LAMBDA programs tested:
  - 5.1. The eight-queen puzzle
  - 5.2. The Eratosthenes's sieve (for constructing streams of primes)
  - 5.3. Some code using the factorial function implemented in Church's
     pure lambda-calculus (Quiz04) to produce the factorial of 5. This
     one is designed to test your compiler's handling of higher-order
     functions
